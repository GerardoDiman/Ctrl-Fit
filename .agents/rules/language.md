# Regla de Idioma: Español Estricto

1. **Uso Obligatorio**: TODA la comunicación con el usuario, la redacción de planes de implementación (implementation_plan.md), tareas (task.md), guías (walkthrough.md), comentarios de código que expliquen la lógica de negocio y cualquier otro artefacto **DEBE ser estrictamente en español**.
2. **Excepciones**: Solo el código en sí mismo (nombres de variables, funciones, nombres de archivos, consultas SQL, etiquetas HTML/JSX) debe mantenerse en inglés para respetar las convenciones de desarrollo de software estándar. Sin embargo, el contenido de la interfaz de usuario (textos visibles para el usuario final) debe ser en español.
3. **Cero Tolerancia**: No se permite escribir planes de implementación ni explicaciones técnicas en inglés. Si se te solicita un plan, piensa y escribe el contenido directamente en español.
